# Daily Logs

这里是默认写作入口。

最适合放：

- 今天做了什么
- 遇到什么问题
- 截图和现象
- AI 给出的启发
- 临时判断和待验证结论

## Start Here

- 新开一天：[[01_Daily_Logs/2026-04-05|Sample Daily]]

## Common Moves

- 某条判断开始稳定：提炼到 `02_Projects/`
- 某个概念以后还会复用：升格到 `03_Knowledge/`
- 某段内容开始要写给别人看：转到 `05_Writing/`

## Best Practice

- 先写事实，不要一边写一边重分类。
- 只要合适，就顺手打上项目、知识、论文或写作页链接。
- 图片保持本地嵌入，交给插件和发布脚本处理。

## How This Layer Connects To Others

- `Daily -> Project`
  - 把重复出现的判断、关键决策、阶段性结论提炼到项目页
- `Daily -> Knowledge`
  - 把跨项目仍然成立的概念、方法、工具做法整理成知识页
- `Daily -> Writing`
  - 当内容开始变成汇报、提纲或段落时，转入写作页重组
- `Daily -> Resources`
  - 如果引用了长对话或原始摘录，只在这里保留摘要，原文链接回来源区

## What Not To Do

- 不要把 Daily 当成最终项目主页
- 不要为了“写得整齐”而推迟记录
- 不要把整篇 Daily 复制到 Project；只提炼稳定判断，必要时附上小范围嵌入
