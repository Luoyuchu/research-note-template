# Knowledge

这里放稳定、可复用、跨项目的内容。

进入这一层之前，最好先问一句：

“如果脱离当前项目，这条内容以后还值得被再次链接吗？”

如果答案是“是”，它更适合进入 Knowledge。

## Start Here

- 理论示例：[[03_Knowledge/Theory/Th Visual Abstraction|Visual Abstraction]]
- 主题示例：[[03_Knowledge/Topics/Top Visual Reasoning|Visual Reasoning]]

## Common Moves

- 某条内容跨项目可复用：放进这里
- 某条内容还离不开项目上下文：先留在 `02_Projects/`
- 某条内容只是原始来源：先留在 `04_Resources/`

## Best Practice

- Knowledge 存的是提炼后的结论，不是原始材料仓库
- 一条知识笔记应该尽量能脱离原始上下文被理解
- 如果某条内容只服务于单一项目且还没稳定，先留在 `02_Projects/`

## How This Layer Connects To Others

- `Daily / Project -> Knowledge`
  - 把经过验证的概念、方法、工具做法抽出来
- `Knowledge -> Project`
  - 项目页反向引用这些稳定结论，避免重复解释
- `Knowledge -> Writing`
  - 写作页可以直接调用这里的理论定义、方法描述、工具工作流
