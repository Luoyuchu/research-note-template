# Writing

这里放面向交付的写作材料。

它回答的问题是：

- 我要写给谁看
- 我要怎么组织论证
- 哪些证据应该进入正文
- 哪些图表和段落需要继续打磨

## Start Here

- 先看一个示例写作页：[[05_Writing/Wri Visual Reasoning Lab Positioning Draft|Positioning Draft]]

## Common Moves

- 先建 outline，再补 section draft
- 证据不复制，尽量链接回 `Project / Knowledge / Papers`
- 如果稿件开始很长，就继续按 deliverable 内部拆页

## Best Practice

- 以 deliverable 为单位组织，而不是镜像整个项目结构
- 保留 outline、section draft、figure plan、revision log 这类写作骨架
- 引用 `Project`、`Knowledge`、`Papers` 和 `Resources`，不要复制它们的全部内容

## How This Layer Connects To Others

- 从 `02_Projects/` 拉取项目事实、实验和决策
- 从 `03_Knowledge/` 拉取概念、方法、工具工作流
- 从 `03_Knowledge/Papers/` 拉取文献依据
- 必要时链接回 `04_Resources/` 中的原始材料

## What Not To Do

- 不要把这里做成第二个项目目录
- 不要完整复制项目流水账
- 不要把所有草稿都堆成无结构长文
