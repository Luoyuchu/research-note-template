# Inbox

这里用来放还没有整理的碎片。

原则很简单：

- 先记下来，不要为了分类中断思路。
- 后续只有在内容真的值得复用时，才转写到其他层。
- 大段 AI 原始对话不要直接散落到知识区，优先放进 `04_Resources/AI_Raw/`。

## Typical Routes

- 今天刚发生的事情：移动或整理进 `01_Daily_Logs/`
- 某个项目的稳定问题或决策：整理进 `02_Projects/`
- 跨项目可复用的概念或方法：整理进 `03_Knowledge/`
- 原始来源材料：整理进 `04_Resources/`
- 已经开始面向交付的提纲或段落：整理进 `05_Writing/`

## Important Reminder

Inbox 不是长期仓库。

- 它负责降阻力捕获
- 不负责最终组织

当前的 `tmp.md` 故意保留了原始图片嵌入，方便测试图片工作流。
