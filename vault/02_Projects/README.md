# Projects

这里放项目主控页和项目级整理结果。

项目页回答的不是“今天发生了什么”，而是：

- 我们在做什么
- 当前最重要的问题是什么
- 做过哪些关键决策
- 下一步推进什么
- 这件事和哪些知识、论文、写作交付有关

## Start Here

- 先看一个示例项目页：[[02_Projects/Prj Visual Reasoning Lab|Visual Reasoning Lab]]

## Common Moves

- 从 Daily 抽出稳定判断：写进项目页
- 需要保留原文证据：引用具体 section / block
- 开始形成交付物：链接到 `05_Writing/`

## Best Practice

- 一个活跃项目通常先有一个 hub note
- 项目页里写结论、判断、当前状态，不写整段流水账
- 如果需要原文证据，优先链接或嵌入某个具体 section / block，而不是复制整篇 Daily

## Suggested Internal Structure

- Goal
- Current Questions
- Current Status
- Decisions
- Related Deliverables
- Evidence Snapshot

## How This Layer Connects To Others

- 来自 `01_Daily_Logs/` 的原始记录先在这里被提炼成项目级判断
- 项目页再链接到 `03_Knowledge/` 中的理论、工具、主题和论文页
- 当需要面向别人表达时，把内容组织到 `05_Writing/`
- 如果底层来源很长，保留在 `04_Resources/`，项目页只引用结论

## What Not To Do

- 不要把项目页做成按日期堆积的日志墙
- 不要在项目页里完整复制写作稿件
- 不要让项目页承担所有原始来源保存职责
